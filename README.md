## About The Project

[![Product Name Screen Shot][product-screenshot]](https://example.com)

This app is an implementation of transactions in mongodb and user interface in react.

For the very first time, when you sign up to the app it generates a random amount of money to your account ranging between 0 to 10000. After sign up, you are directed to dashboard where there is a list of users, also you can search for users in the database. Against every user there is a send money button. On clicking it takes you to a send money page where you are supposed to enter the amount and initiate the payment.

Further advancements in the app will be modified.

## Getting Started

To set this project locally, do the following steps: 

### Prerequisites

You should have node.js locally on your computer.

 
 **Node.js Installation Tutorials**

- [Node.js Installation for Mac](https://nodejs.org/en/download/package-manager/#macos)
- [Node.js Installation for Windows](https://nodejs.org/en/download/package-manager/#windows)
- [Node.js Installation for Linux](https://nodejs.org/en/download/package-manager/#debian-and-ubuntu-based-linux-distributions)

### Installation

Below are the steps you need to follow to successfully run the project locally at your side :

1. Clone the repository:
    ```sh
    git clone https://github.com/abhijeetmishra2104/quickFunds.git
    ```

2. Navigate to the project directory:
    ```sh
    cd quickFunds
    ```

3. Navigate to the backend directory and install dependencies:
    ```sh
    cd backend
    npm install
    ```

4. Navigate to the frontend directory and install dependencies:
    ```sh
    cd ../frontend
    npm install
    ```

5. Start the backend server:
    ```sh
    cd ../backend
    npm start
    ```

6. In a new terminal window, navigate to the frontend directory and start the frontend server:
    ```sh
    cd ../frontend
    npm start
    ```

### Explanation

1. **Clone the repository**: Fetches the project from GitHub.
2. **Navigate to the project directory**: Changes into the main project folder.
3. **Navigate to the backend directory and install dependencies**: Installs backend dependencies.
4. **Navigate to the frontend directory and install dependencies**: Installs frontend dependencies.
5. **Start the backend server**: Runs the backend server.
6. **Start the frontend server**: Runs the frontend development server in a new terminal window.

This setup will start both the backend and frontend servers, allowing you to run and test the application.

## Contributing

I welcome contributions to improve and expand this project! Here are some guidelines to help you get started:

### How to Contribute

1. **Fork the Repository**: Click the "Fork" button at the top right of this page to create a copy of the repository on your GitHub account.

2. **Clone Your Fork**: Clone your forked repository to your local machine.
    ```sh
    git clone https://github.com/abhijeetmishra2104/quickFunds.git
    cd quickFunds
    ```

3. **Create a Branch**: Create a new branch for your feature or bugfix.
    ```sh
    git checkout -b feature/your-feature-name
    ```

4. **Make Changes**: Make your changes in the codebase. Ensure that your code adheres to the project's coding standards and passes all tests.

5. **Commit Your Changes**: Commit your changes with a clear and descriptive commit message.
    ```sh
    git commit -m "Add feature: your feature description"
    ```

6. **Push to Your Fork**: Push your changes to your forked repository.
    ```sh
    git push origin feature/your-feature-name
    ```

7. **Open a Pull Request**: Open a pull request on the original repository. Provide a clear description of the changes you made and any additional context or information that might be helpful for the reviewers.
